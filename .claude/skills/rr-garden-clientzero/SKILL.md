# RR Garden as Client Zero

Written 2026-07-12. Ground truth: `context/rr-garden.md`, `goal.md` Constraints,
interview Q7 (July 2026).

## What RR Garden is (zero-context)
Atishay's father's guest house + open-lawn wedding venue in Kanpur, carrying a ~₹1 crore
property loan (father's obligation, NOT Atishay's). Family expects Atishay to help grow
it. The project treats it as **Client Zero**: the practice client where systems get
proven — under a **hard 6–8 hours/week cap**.

## The physical reality (confirmed July 2026 — quote these, don't embellish)
4 operable rooms today (of 12 planned; first floor under construction); OYO-listed,
1–2 bookings/day at ₹400–800 (to ~₹1,000 night) → rooms gross ceiling roughly ₹20–40K/mo
as-is (labeled inference from those figures); ~6–10 weddings/functions last year; event
capacity ~700–1,000 guests; staff = ONE semi-manager (~17, turns 18 this year, lives
on-site, ₹20K + ₹50/booking); Instagram page + Google Business Profile + OYO listing
exist but weak.

## Father's ACTUAL ask (his stated expectation — the scope)
Marketing and growth ONLY: make videos, promote, run Meta Ads targeted Kanpur + Lucknow
for bookings. Physical presence NOT required. Later phase: restaurant + off-season events
(kitty parties, birthdays, private screenings). Anything outside this scope is scope
creep — decline or renegotiate.

## The work, in priority order
1. **Day-30 written agreement with father** (~by Aug 10, 2026): role scope, authority to
   test offers/spend, data access, compensation-if-profit-grows. Without it, this becomes
   unpaid family labour — the archive doc's trap #1.
2. **Baseline audit** (same deadline): enquiries, bookings, sources, revenue, expenses —
   currently UNDOCUMENTED; no growth claim is possible without it.
3. **Enquiry system build** (weeks 3–6): WhatsApp Business auto-reply → enquiry log →
   follow-up cadence → source tracking. Build it so the semi-manager runs it from a
   checklist. This doubles as the growth-systems demo + first case study.
4. **Content + Meta Ads**: venue videos on RR Garden's OWN page (client bar, not
   HybridBanda bar); ad budget agreed with father BEFORE spending, spend separate from
   any fee; first campaigns = off-season event offers for the new hall.

## "Done correctly" looks like
Hours logged ≤8/week; every ad/video routes to the tracked WhatsApp system; father sees
a monthly one-page report; ads never promise more than 4 rooms can deliver.

## Boundary rules (non-negotiable)
Cap exceeded 3 consecutive weeks → renegotiate scope with father, never silently absorb.
Paint shop: zero hours this year. Wedding-media agency: PARKED — 6–10 weddings/yr is
below even the archive's weak validation case; only revisit after ≥5 packages sell via
partner vendors (see failure-archaeology).

## When NOT to use this skill
Not for external paying clients (they outrank RR Garden in the priority stack —
decision-rules skill). Not for family finance/loan matters beyond the written agreement —
that's the father's domain.

How to re-verify: check the Day-30 agreement + baseline exist; if either is missing after
Aug 10, 2026, they are the top overdue RR Garden items.
